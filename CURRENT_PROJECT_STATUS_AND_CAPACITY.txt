4) CAPACITY PROFILE (PRACTICAL, CURRENT ARCHITECTURE)
==================================================
This is a local-first architecture. Capacity depends on browser memory,
local CPU/GPU, and Ollama model throughput.

Current observed workload:
- 1,121 orders handled in Deep Dive
- 214 chunks indexed
- Local embedding and retrieval flow remains responsive at this size

Practical operating envelope (without backend scaling changes):
- Small to medium datasets (hundreds to a few thousand orders): GOOD
- Tens of thousands of orders in browser-only cache: RISKY
  - localStorage size pressure
  - slower index build and retrieval latency
  - larger prompt context management overhead

LLM throughput constraints:
- Bound by local Ollama host hardware (CPU/GPU and RAM)
- Multi-user concurrency is limited in single-machine local setup
- Suitable for single-operator or small-team usage

Proxy capacity constraints:
- Single Node process with timeout-based upstream calls
- Good for local/dev and moderate request volume
- Not yet production-hardened for high concurrent traffic
